2、MySQL中的列类型
创建数据表的时候，指定的列类型可以存储 的数据类型。
CREATE TABLE book（bid INT（列类型））；
（1） 数值类型（数值型的引号可以省略）
Tinyint 微整型7、占1个字节 范围是：-128—127
Smallint 小整型15、占2个字节 范围是：-32768—32767
INT 整型32、占4个字节 范围是：-2147483648—2147483647
BIGINT 大整型 占8个字节
FIOAT 单精度浮点型，占4个字节，范围3.4e38，范围比INT大很多，可能产生计算误差。（精确度没有INT高）
DOUBLE 双精度浮点型，占8个字节，范围比BIGINT大的多
DECIMAL(M,D)定点小数，不会产生计算误差，M代表总的有效位数，D代表小数点后的有效位数
BOOL：布尔型，只有两个结果TRUE/1、FALSE/0,用于只有两种情况的布局(布尔型TRUE和FALSE不需要加引号)；正在存储数据的时候，会使用TINYINT。
（2） 日期时间类型（必须添加引号）
DATE 日期型 ‘2018-12-32’
TIME 时间型 ‘14:22:30’
DATETIME 日期时间型 ‘2018-12-31 14:22:30’
（3） 字符串类型（必须添加引号）
VARCHAR（M）:变长字符串 m指的是参数，不会产生空间浪费，操作速度相对较慢，M最大值是65535
CHAR（M）：定长字符串，可能产生空间浪费，但操作速度比较快，M最大值是255；用于存储一个人的手机号码、身份证号等固定长度的字符。
TEXT(M)大型变长字符串，最多存2G。

3、列约束
MySQL可以对插入的数据进行特定的验证，只有满足条件才允许插入到数据表中，否则被认为是非法的插入。
例如：一个人的性别只能是男或者女，一个人的年龄0到100之间。
（1） 主键约束 PRIMARY KEY,
声明了主键列上的值不能出现重复，表中查询的记录会按照主键由小到大排序——加快查找的速度；通常主键添加到编号列中。
注意：一个表中只能有一个主键
（2） 唯一约束 UNIQUE（唯一的）,NULL不允许加引号
声明了唯一约束的列上不能插入重复的值，允许插入NULL，而且允许插入多个NULL。
NULL表示空，在插入数据，暂时无法确定要保存的数据。
如：无法确定员工的姓名、工资。
（3） 非空约束 NOT NULL
声明了非空约束的列上不能插入NULL
（4） 默认值约束 DEFAULT
default ‘2016-6-6’
可以使用DEFAULT关键字声明默认值，有两种方式可以应用默认值
INSERT INTO xz_laptop_family VALUES(40,’苹果’,DEFAULT );
INSERT INTO xz_laptop_family（fid,fname） VALUES(50,’华硕’);
（5） 检查约束 CHECK
检查约束可以对插入的数据进行检验
CREATE TABLE student（
age TINYINT CHECK（age>=18 AND age<=60）
）;
MySQL不支持检查约束，会降低数据的插入速度。
（6） 外键约束 FOREIGN KEY
声明了外键约束的列，取值必须在另一个表的主键列上出现过，列类型要保存一致，取值可以是NULL
FOREIGN KEY(familyld)
REFERENCES xz_laptop_family(fid)

2、MySQL中的自增列
AUTO_INCREMENT:自动增长，假如一个列声明了自增列，无需手动赋值，直接设置为NULL，会获取当前的最大值，然后加1插入。
注意：
自增列允许手动赋值
只适用于整数型的主键列上
INSERT INTO xz_laptop VALUES(NULL,’苹果’,4000,’14.6’,’超越介绍’,’2018-9-6’,1,20 );
